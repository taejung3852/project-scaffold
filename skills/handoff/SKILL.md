# /handoff — 세션 인계 스킬

지금까지의 대화와 작업 컨텍스트를 압축해 저장하고, 다음 세션(또는 다른 에이전트)이 그대로 이어받을 수 있게 한다.
`/clear`나 새 세션 시작으로 컨텍스트가 끊겨도 처음부터 다시 설명할 필요가 없다.

---

> **추적:** `python3 scripts/skill_usage.py track handoff`

---

## Step 0 — 모드 결정

`/handoff` 실행 시 모드를 확인한다.

| 모드 | 트리거 | 동작 |
|---|---|---|
| `save` | `/handoff save [다음 세션 목표]` 또는 세션 종료 의도 표현 시 | 현재까지 대화·작업 압축 저장 |
| `load` | `/handoff load` | 가장 최근 저장본 복원 |
| `list` | `/handoff list` | 저장된 인계 목록 표시 |
| `clean` | `/handoff clean [N일]` | N일(기본 30일) 이상 지난 저장본 삭제 |

인자가 없으면 묻는다:

> "세션을 저장할까(save), 이전 세션을 불러올까(load)?"

인자로 텍스트가 주어지면(`/handoff save 다음엔 인증 모듈 테스트 작성`) 다음 세션이 무엇에 집중할지에 대한 설명으로 간주하고, 문서 내용을 그 목표에 맞춰 작성한다.

---

## Step 1 — save

### 1-1. 대화 요약

지금까지의 대화에서 다음을 정리한다:

- 무엇을 하려고 시작했는지
- 무엇을 완료했는지
- 진행 중이던 작업과 그 상태
- 막힌 부분·미해결 질문
- 다음 세션이 집중할 목표 (인자로 받았으면 그걸 우선, 없으면 자연스러운 다음 단계 추론)

**이미 다른 산출물(PR, 커밋, 결정 문서, 이슈, diff)에 적혀 있는 내용은 다시 풀어 쓰지 않는다.** 경로나 URL로 참조한다.

### 1-2. git 컨텍스트 (보조 정보)

```bash
git branch --show-current
git log -5 --oneline
git status --short
```

### 1-3. 추천 스킬

다음 세션에서 바로 실행하면 좋을 스킬을 1~3개 제안한다 (예: `raw/`에 pending 파일이 있으면 `/ingest`, PR 올리기 직전이면 `/code-lint`).

### 1-4. 민감 정보 검열

API 키, 토큰, 비밀번호, 개인정보 등이 대화나 git 출력에 포함돼 있으면 `[REDACTED]`로 가린다.

### 1-5. 저장

`raw/handoffs/YYYY-MM-DD_HHmm.md` 에 저장한다:

```markdown
---
date: YYYY-MM-DD HH:mm
branch: [브랜치명]
---

## 다음 세션 목표
[인자로 받은 목표, 없으면 추론한 다음 단계]

## 지금까지
[무엇을 하려 했고, 무엇을 완료했는지]

## 진행 중 / 막힌 부분
[미완료 작업, 미해결 질문]

## 관련 산출물
- [PR/커밋/문서 경로나 URL — 내용 중복 금지, 참조만]

## 추천 스킬
- `/스킬명` — [이유]
```

> 이 파일은 `ingest_status` 필드를 쓰지 않는다 — `/ingest` 대상이 아니다. 세션 운영 기록일 뿐 wiki에 통합할 지식이 아니다.

저장 후 채팅에 같은 내용을 요약해 출력한다.

---

## Step 2 — load

`raw/handoffs/` 에서 가장 최근 파일을 읽는다 (날짜가 인자로 주어지면 해당 파일).

읽은 내용을 그대로 정리해 출력하고, 저장 시점 이후 git 상태가 달라졌으면 알려준다:

```bash
git status --short
```

> "저장 시점 이후 변경된 파일이 있어: [목록]"

---

## Step 3 — list

`raw/handoffs/` 파일들을 날짜 역순으로, 각 파일의 "다음 세션 목표" 한 줄과 함께 나열한다.

---

## Step 4 — clean

기본 30일(인자로 지정 시 해당 일수) 이상 지난 파일 목록을 보여주고 삭제 확인을 받는다. **확인 없이 삭제하지 않는다.**

---

## 규칙

| 항목 | 규칙 |
|---|---|
| 저장 위치 | `raw/handoffs/` — `/ingest` 대상 아님 (`ingest_status` 필드 생략) |
| 중복 금지 | PR·커밋·결정 문서에 이미 있는 내용은 재서술하지 않고 경로/URL로만 참조 |
| 민감 정보 | API 키·토큰·개인정보는 저장 전 반드시 검열 |
| load 시 git 상태 비교 | 저장 시점과 달라진 게 있으면 반드시 알림 |
| 삭제(clean) | 항상 사용자 확인 후 진행 |
