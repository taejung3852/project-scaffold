# /report — 리포트 생성 스킬

회의·인터뷰·스프린트 등의 내용을 받아 목적에 맞는 문서를 생성한다.

---

> **추적:** `python3 scripts/skill_usage.py track report`

---

## Step 0 — 리포트 타입 선택

```
1) 회의록        — 팀 정기 회의, 기획 회의
2) 인터뷰 리포트 — 사용자·고객·팀원 딥인터뷰
3) ADR           — Architecture Decision Record, 기술 결정 문서
4) 스프린트 요약 — 스프린트 회고·계획
```

인자로 타입이 명시된 경우(`/report 인터뷰`) 바로 해당 타입으로 진행한다.

---

## Step 1 — 원본 내용 수집

**인자가 파일 경로인 경우:** 해당 파일을 읽는다.

**인자가 없거나 타입만 있는 경우:**

> "리포트로 만들 내용을 붙여넣어 주세요. 완료되면 END 입력."

붙여넣은 내용을 그대로 사용한다. 내용을 수정하거나 요약하지 않는다.

---

## Step 2 — 출력 모드 선택

> "어떤 용도로 만들까요?
> 1) 내 파악용 — 상세 인사이트·의문점·주관적 해석 포함
> 2) 팀 공유용 — 결정사항·액션아이템·담당자 중심의 공유 문서
> 3) 둘 다 — 두 버전 동시 생성"

---

## Step 3 — 리포트 생성

### 내 파악용 포맷

```markdown
# [제목] — 내부 메모 (YYYY-MM-DD)

## 1분 요약
(핵심 3줄 이내)

## 상세 내용
### 논의된 주제들
(항목별 정리)

### 내 해석 & 인사이트
- 이 결정이 [X]에 미치는 영향:
- 확인이 더 필요한 사항:
- 아직 해결 안 된 의문:

### 주목할 인용
> "..." — [발화자]

## 다음에 알아볼 것들
- [ ] ...
```

### 팀 공유용 포맷

```markdown
# [제목] (YYYY-MM-DD)

**참석자:** ...
**작성자:** ...

## 결정사항
| # | 결정 내용 | 배경 |
|---|---|---|
| 1 | | |

## 액션아이템
| 항목 | 담당자 | 기한 |
|---|---|---|
| | | |

## 논의 요약
(회의 흐름 중심으로 간결하게)

## 다음 미팅
- 일시:
- 안건:
```

### ADR 포맷 (타입 3 선택 시)

```markdown
# ADR-NNN: [제목]

**상태:** proposed | accepted | deprecated
**날짜:** YYYY-MM-DD
**결정자:** ...

## 맥락
(이 결정이 필요했던 배경·제약·문제)

## 결정
(우리가 선택한 것과 이유)

## 결과
**긍정적:**
**부정적:**
**중립적·트레이드오프:**
```

### 스프린트 요약 포맷 (타입 4 선택 시)

```markdown
# 스프린트 [N] 요약 (YYYY-MM-DD ~ YYYY-MM-DD)

## 완료
- [x] ...

## 미완료 & 이유
- [ ] ... — 이유:

## 회고
**잘 된 것:**
**개선할 것:**

## 다음 스프린트 목표
1. ...
```

---

## Step 4 — 저장 및 출력

| 용도 | 저장 경로 |
|---|---|
| 내 파악용 | `wiki/meetings/YYYY-MM-DD_[slug]_internal.md` |
| 팀 공유용 | `wiki/meetings/YYYY-MM-DD_[slug]_shared.md` |

저장 후:
- 저장 경로 출력
- **팀 공유용**은 마크다운 블록으로도 바로 출력 (Slack·Notion에 복붙 가능하도록)
- **둘 다**인 경우 두 파일 모두 저장 후 공유용만 블록 출력

---

## 규칙

| 항목 | 규칙 |
|---|---|
| 원본 내용 | 수정·삭제 금지. 리포트는 재구성이지 편집이 아님 |
| 사실 vs 해석 | 내 파악용에서만 해석 허용. 팀 공유용은 사실과 결정만 |
| 담당자 불명확 시 | `TBD`로 표시하고 괄호에 후보 명시 |
| 기한 불명확 시 | `미정`으로 표시 (임의 날짜 생성 금지) |
| ADR 번호 | `wiki/meetings/` 내 기존 ADR 파일 수 + 1로 자동 부여 |
