# unclear-path — UNCLEAR 경로 상세 절차

`ambiguity-check.md`에서 **UNCLEAR**로 분류됐거나,
CLEAR 경로에서 follow-up 2회 초과 후에도 모호한 항목의 처리 절차.

---

## 원칙

질문하지 않는다. **연구(스캔 결과·기본값)로 해결한다.**

---

## 처리 순서

```
1. Step 0.5 스캔 결과에 해당 항목이 있나?
   → YES: 스캔 결과를 기본값으로 채택

2. 없으면: 해당 카테고리의 업계 표준 기본값 채택
   예) 테스트 커버리지 미결정 → "80% (업계 일반 기준)"

3. 기본값 설정 후 다음 형식으로 한 줄 기록:
   "(기본값) [항목]: [값] — 나중에 /setup으로 바꿀 수 있어."

4. 여전히 기본값 선택 불가 (irreversible·보안·위험 항목):
   딱 하나의 집중 질문만 허용.
   "이것만 결정해줘: [핵심 선택지 A vs B]"
```

---

## TBD 처리 (최후 수단)

기본값도 없고, 집중 질문에도 답이 없는 경우:

> "지금은 `TBD`로 표시하고 넘어갈게. 나중에 `/setup`으로 이 항목만 다시 정할 수 있어."

`wiki/conventions/[해당카테고리].md`에 해당 항목을 `TBD`로 기록한다.

---

## 기록 형식

```markdown
| 항목 | 값 | 출처 |
|---|---|---|
| 테스트 커버리지 | 80% | 기본값 (업계 기준) |
| 브랜치 전략 | Git Flow | Step 0.5 스캔 감지 |
| PR merge 방식 | TBD | 미결정 |
```
