# /devlog — 세션 기반 dev-log 스킬

`/capture dev-log`가 질문만 던지는 것과 달리, 이 스킬은 세션을 먼저 분석해 초안을 만들고 확인·보완만 받는다.

---

> **추적:** `python3 scripts/skill_usage.py track devlog`

---

## Step 0 — 컨텍스트 자동 수집

다음을 병렬로 수집한다:

```bash
git log --since="00:00" --oneline          # 오늘 커밋
git diff HEAD~1 --stat 2>/dev/null         # 변경 파일 목록
git status --short                          # 현재 상태
```

대화 컨텍스트(현재 세션에서 오간 내용)도 함께 참조한다.

---

## Step 1 — Claude가 초안 생성

수집한 컨텍스트를 바탕으로 다음 5개 필드를 먼저 채운다:

| 필드 | 자동 추론 방법 |
|---|---|
| 제목 | 오늘 커밋 메시지 + 주요 변경 파일에서 추출 |
| 오늘 한 일 | 커밋 메시지 + 변경 파일 목록 기반 서술 |
| 막혔던 부분 | 대화에서 "왜", "에러", "안 돼", "해결" 키워드 감지 |
| 다음 할 일 | 대화의 미완 항목, TODO, 열린 질문 추출 |
| 배운 것 | 대화에서 새로운 발견·결정·인사이트 추출 |

추론이 불가능한 필드는 빈 칸으로 두고 Step 2에서 확인한다.

---

## Step 2 — 확인·보완 (추천값 포함)

초안을 보여주며 각 필드를 한 번에 확인한다. 형식:

```
[devlog 초안]

📌 제목: init.sh UX 개선 — 경로 입력 필수화
✅ 오늘 한 일: init.sh 수정 (경로 입력 필수화, raw/ gitignore 추가), skills/setup/SKILL.md 업데이트
❓ 막혔던 부분: (감지 못했어. 있었어?)
✅ 다음 할 일: devlog 스킬 추가, README 업데이트
❓ 배운 것: (있으면 추가해줘)

수정할 내용 있으면 말해줘. 없으면 그대로 저장할게.
```

- `✅` — 자신 있게 추론된 항목
- `❓` — 확인 필요한 항목만 질문

사용자가 "없어" / "그냥 저장해" 하면 빈 칸 그대로 저장한다. 억지로 채우지 않는다.

---

## Step 3 — 파일 생성

**저장 경로:** `raw/dev-logs/YYYY-MM-DD_dev-log_슬러그.md`

**슬러그 규칙:** 제목에서 추출, 공백 → `-`, 특수문자 제거, 최대 40자

**frontmatter 템플릿:**

```yaml
---
title: "[제목]"
raw_type: "dev-log"
date: YYYY-MM-DD
created: YYYY-MM-DD
ingest_status: "⏳ pending"
tags:
  - "raw/dev-log"
---
```

당일 파일이 이미 존재하면 덮어쓰지 않고 `_2`, `_3` suffix를 붙인다.

---

## Step 4 — 완료 안내

```
저장 완료: raw/dev-logs/2026-06-19_dev-log_init-sh-ux.md

바로 wiki에 반영할까? (y/n)
```

- **y** → `/ingest [파일경로]` 실행
- **n** → "나중에 `/ingest`로 처리하세요." 안내

---

## 규칙

| 항목 | 규칙 |
|---|---|
| 컨텍스트 우선 | 질문 전에 반드시 git log + 대화 먼저 읽기 |
| 추천값 제시 | 확인 질문엔 항상 Claude 추론값을 함께 보여줌 |
| 빈 칸 허용 | 사용자가 건너뛰면 해당 필드 생략하고 저장 |
| 원문 유지 | 사용자 답변은 편집 없이 그대로 기록 |
| 덮어쓰기 금지 | 동일 날짜 파일 존재 시 suffix 붙이기 |
