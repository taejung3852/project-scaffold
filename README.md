# project-scaffold

> GitHub template for setting up an LLM Wiki-based development harness — conventions, devlog, dashboard, and agent setup in one place.

## What is this?

A GitHub template that wires up a full **LLM Wiki + agent harness** for any software development project.

Fork it, run `/setup`, and get:

- **Convention Wiki** — team conventions defined via deep interview, stored as living wiki pages
- **Auto devlog** — triggered on every git commit + daily summary
- **Dashboard** — priority task list from meeting captures, rendered as markdown → terminal → web UI
- **TDD enforcement** — agent is instructed to write tests before implementation
- **Code lint** — LLM + static analysis (pylint/eslint) checks code against your convention wiki
- **Wiki lint** — keeps your project wiki clean and consistent
- **Agent-agnostic** — `AGENT.md` as single source of truth, works with Claude Code, Cursor, Windsurf, and more

## Quick Start

```bash
# 1. Use this template on GitHub
# 2. Clone your new repo
# 3. Open in your coding agent and run:
/setup
```

The `/setup` skill will interview you about your project (goals, tech stack, conventions, TDD rules, devlog format, risk criteria) and generate your `wiki/conventions/` automatically.

## Project Structure

```
wiki/
  conventions/     ← generated by /setup interview
  meetings/        ← captured meeting notes
  devlog/          ← auto-generated per commit + daily
  dashboard.md     ← auto-updated priority list
AGENT.md           ← generated by /setup (agent instructions)
CLAUDE.md          ← @AGENT.md (one-liner for Claude Code)
skills/
  setup/           ← deep interview → convention wiki
  capture/         ← meeting capture
  dashboard/       ← dashboard rendering
  wiki-lint/       ← wiki quality check
  code-lint/       ← convention-based code validation
.hooks/            ← convention enforcement hooks
.obsidian/         ← pre-configured graph view + folder colors
```

## Philosophy

Built on the [LLM Wiki pattern](https://x.com/karpathy) — instead of RAG, the agent owns and maintains a markdown wiki that compounds knowledge over time.

The convention wiki is the single source of truth. The hook reads it. The agent follows it. The devlog reflects it.
